## 摘要
计算机使用智能体 (CUAs) 利用大语言模型在桌面环境中执行图形用户界面 (GUI) 操作，但它们在生成动作时往往不评估动作质量，导致不可逆的错误在后续步骤中产生级联效应。我们提出了 IntentScore，这是一种感知计划的奖励模型，它通过学习跨越三个操作系统的 39.8 万个离线 GUI 交互步骤来对候选动作进行评分。IntentScore 采用两个互补的目标进行训练：用于状态-动作相关性的对比对齐，以及用于动作正确性的边际排名。在架构上，它将每个候选动作的规划意图嵌入到动作编码器中，从而能够区分动作相似但逻辑依据不同的候选动作。IntentScore 在留出法评估中实现了 97.5% 的成对判别准确率。作为 Agent S3 在 OSWorld（一个训练期间完全未见的环境）上的重排序器部署时，IntentScore 将任务成功率提高了 6.9 个百分点，证明了从异构离线轨迹中学习到的奖励估计可以泛化到未见的智能体和任务分布。

## Abstract
Computer-Use Agents (CUAs) leverage large language models to execute GUI operations on desktop environments, yet they generate actions without evaluating action quality, leading to irreversible errors that cascade through subsequent steps. We propose IntentScore, a plan-aware reward model that learns to score candidate actions from 398K offline GUI interaction steps spanning three operating systems. IntentScore trains with two complementary objectives: contrastive alignment for state-action relevance and margin ranking for action correctness. Architecturally, it embeds each candidate's planning intent in the action encoder, enabling discrimination between candidates with similar actions but different rationales. IntentScore achieves 97.5% pairwise discrimination accuracy on held-out evaluation. Deployed as a re-ranker for Agent S3 on OSWorld, an environment entirely unseen during training, IntentScore improves task success rate by 6.9 points, demonstrating that reward estimation learned from heterogeneous offline trajectories generalizes to unseen agents and task distributions.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：当前的计算机使用智能体（CUAs）通常采用“单次调用生成单次动作”的模式。由于缺乏对生成动作质量的评估机制，一旦智能体在复杂的图形用户界面（GUI）中做出错误决策（如误删文件、点击错误按钮），这些错误会迅速累积且难以恢复。
*   **研究背景**：虽然可以通过采样多个候选动作并让 LLM 自我评估来缓解，但这会带来双倍的推理成本和自我偏好偏差。因此，需要一个轻量级、独立训练且具备跨系统泛化能力的奖励模型，来识别“看似合理但实际错误”的动作。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：利用大规模多操作系统（Windows, Mac, Ubuntu）的离线交互轨迹，训练一个**感知计划的双编码器奖励模型**。该模型不仅看“做了什么”，还看“为什么这么做”（即意图）。
*   **关键技术细节**：
    *   **意图感知编码（Intent-Conditioned）**：将 LLM 生成的“思考过程（Thought）”嵌入到动作编码器中。这使得模型能区分两个物理坐标相同但动机不同的动作。
    *   **双目标训练框架**：
        1.  **对比对齐（InfoNCE Loss）**：学习状态与动作的相关性，即“在这个界面下，这个动作是否相关”。
        2.  **边际排名（Margin Ranking Loss）**：针对“硬负样本”（如时序上相邻但当前错误的动作）进行训练，强制拉开正确动作与错误动作的得分差距。
    *   **两阶段流水线**：先在 398K 个跨 OS 步骤上进行预训练，学习通用的 GUI 交互模式；再在目标域（Ubuntu）进行微调。
*   **算法流程**：智能体针对当前状态生成 $N$ 个候选动作 $\rightarrow$ IntentScore 分别计算状态向量与各动作向量的余弦相似度 $\rightarrow$ 选择得分最高且超过阈值的动作执行。

### 6. 主要结论与发现
*   **性能提升**：IntentScore 在离线辨别任务中达到 **97.5%** 的准确率；在 OSWorld 在线测试中，将 Agent S3 的成功率从 45.2% 提升至 **52.1%**（+6.9 pts）。

*   **轻量且高效**：13M 参数量使其几乎不增加推理延迟，远优于调用大模型进行重排序。
*   **意图感知**：敏锐地捕捉到“意图（Thought）”在区分相似 GUI 动作中的关键作用。
*   **泛化能力强**：在训练集完全未包含的 OSWorld 环境中表现稳健，证明了其作为通用奖励模型的潜力。

### 8. 不足与局限
*   **分布偏移（Distribution Shift）**：离线训练数据（AgentNet）与在线部署智能体（Agent S3）生成的动作在代码风格和规划格式上存在差异，限制了性能的进一步释放。
*   **单步评估限制**：目前仅进行单步动作评估，缺乏对多步长远规划的价值估计（Value Estimation）。
*   **视觉瓶颈**：由于使用了冻结的视觉编码器（SigLIP），在处理极小 UI 元素或需要精细视觉反馈的任务（如 Impress）中表现受限。
