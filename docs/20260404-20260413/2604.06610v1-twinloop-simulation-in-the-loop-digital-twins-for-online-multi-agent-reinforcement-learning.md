## 摘要
分布式在线学习使信息物理多智能体系统能够实现运行时自适应，但当运行条件发生变化时，学习到的策略通常需要大量的试错交互才能恢复性能。为解决这一问题，我们提出了 TwinLoop，这是一种用于在线多智能体强化学习的仿真在环数字孪生框架。当发生上下文偏移时，数字孪生被触发以重建当前系统状态，利用最新的智能体策略进行初始化，并通过仿真假设分析（what-if analysis）执行加速策略改进，最后将更新后的参数同步回物理系统中的智能体。我们在工作负载和基础设施条件不断变化的车辆边缘计算任务卸载场景中评估了 TwinLoop。结果表明，数字孪生可以提高偏移后的自适应效率，并减少对昂贵的在线试错的依赖。

## Abstract
Decentralised online learning enables runtime adaptation in cyber-physical multi-agent systems, but when operating conditions change, learned policies often require substantial trial-and-error interaction before recovering performance. To address this, we propose TwinLoop, a simulation-in-the-loop digital twin framework for online multi-agent reinforcement learning. When a context shift occurs, the digital twin is triggered to reconstruct the current system state, initialise from the latest agent policies, and perform accelerated policy improvement with simulation what-if analysis before synchronising updated parameters back to the agents in the physical system. We evaluate TwinLoop in a vehicular edge computing task-offloading scenario with changing workload and infrastructure conditions. The results suggest that digital twins can improve post-shift adaptation efficiency and reduce reliance on costly online trial-and-error.

### 2. 方法论：核心思想与技术细节
TwinLoop 框架的核心思想是**“物理感知-虚拟演练-知识同步”**，具体分为三个阶段：
*   **阶段 1：快照（Snapshot）**：当触发机制启动时，DT 获取物理系统（PT）的当前状态快照，包括服务器处理能力、任务队列、车辆位置、任务到达率以及所有智能体的神经网络权重。
*   **阶段 2：DT 训练（DT Training）**：在数字孪生中重建环境，并以物理智能体的最新策略为起点。利用**加速仿真**（比现实快 25 倍）进行“假设分析”（What-if analysis）。智能体采用 **Dueling Double DQN (D3QN)** 算法，并配合 **Boltzmann 探索策略**（通过调节温度参数 $\tau$ 在仿真中快速探索新策略）。
*   **阶段 3：权重同步（Weight Synchronisation）**：仿真优化完成后，将更新后的网络权重同步回物理系统中的智能体，使其能够立即以更优的策略应对新环境。

### 6. 主要结论与发现
*   **加速收敛**：TwinLoop 显著缩短了环境变化后的性能恢复时间，尤其在环境压力较大的阶段（阶段 1 和 2），平均延迟和尾部延迟均大幅下降。
*   **触发频率的权衡**：增加 DT 触发频率在一定程度上有利于适应，但过于频繁的更新（如 $k=4$）反而会因策略波动导致性能下降。
*   **DT 的价值**：数字孪生不仅提供了无成本的探索空间，还通过加速仿真让智能体在物理环境受到影响前就“预见”了最优策略。

### 7. 优点
*   **实用性强**：针对在线学习“试错成本高”的痛点，提出了闭环的数字孪生解决方案。
*   **仿真加速**：利用 25 倍的仿真加速比，实现了真正意义上的“运行时”策略优化。
*   **鲁棒性提升**：实验证明该方法能有效抑制环境偏移带来的延迟激增（P99 延迟显著降低）。

### 8. 不足与局限
*   **快照一致性假设**：论文假设可以获取全局一致的物理系统快照，但在大规模分布式系统中，实时同步所有智能体状态和权重具有极高的通信开销。
*   **触发机制简单**：目前采用固定频率触发，而非根据环境变化的检测结果动态触发。
*   **场景单一**：实验环境为同质化的城市网格，对于高度异构（如城乡结合、复杂拓扑）的网络环境，多场景训练的优势尚未完全体现。
