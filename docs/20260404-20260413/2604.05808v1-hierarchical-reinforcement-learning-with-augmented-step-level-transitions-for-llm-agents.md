## 摘要
大语言模型（LLM）智能体在复杂的交互式决策任务中展现出了强大的能力。然而，现有的 LLM 智能体通常依赖于日益增长的交互历史，导致计算成本高昂且可扩展性有限。在本文中，我们提出了 STEP-HRL，这是一种分层强化学习（HRL）框架，它通过仅以单步转移而非完整交互历史为条件，实现了步级学习。STEP-HRL 对任务进行分层构建，利用已完成的子任务来表示整个任务的全局进度。通过引入局部进度模块，它还迭代且有选择性地总结每个子任务内的交互历史，从而生成局部进度的紧凑摘要。这些组件共同为高层和底层策略生成了增强的步级转移。在 ScienceWorld 和 ALFWorld 基准测试上的实验结果一致表明，STEP-HRL 在性能和泛化能力方面显著优于基准模型，同时减少了 token 使用量。我们的代码可在 https://github.com/TonyStark042/STEP-HRL 获取。

## Abstract
Large language model (LLM) agents have demonstrated strong capabilities in complex interactive decision-making tasks. However, existing LLM agents typically rely on increasingly long interaction histories, resulting in high computational cost and limited scalability. In this paper, we propose STEP-HRL, a hierarchical reinforcement learning (HRL) framework that enables step-level learning by conditioning only on single-step transitions rather than full interaction histories. STEP-HRL structures tasks hierarchically, using completed subtasks to represent global progress of overall task. By introducing a local progress module, it also iteratively and selectively summarizes interaction history within each subtask to produce a compact summary of local progress. Together, these components yield augmented step-level transitions for both high-level and low-level policies. Experimental results on ScienceWorld and ALFWorld benchmarks consistently demonstrate that STEP-HRL substantially outperforms baselines in terms of performance and generalization while reducing token usage. Our code is available at https://github.com/TonyStark042/STEP-HRL.

### 1. 核心问题与研究动机
*   **核心问题**：现有的 LLM 智能体在处理长程决策任务时，通常将整个交互历史（Observations + Actions）作为上下文输入。这种“历史依赖型”设计导致了两个主要问题：
    1.  **计算成本高昂**：随着步数增加，Token 消耗呈二次方增长。
    2.  **推理干扰**：冗长且未经过滤的历史包含大量冗余信息，会掩盖关键决策信号，导致模型推理质量下降。
*   **研究背景**：虽然分层强化学习（HRL）已被引入 LLM 领域，但现有方法仍让高层和底层策略依赖完整历史。本文旨在探索一种**步级（Step-level）**的决策模式，即仅依赖当前状态的紧凑表示进行决策。

### 2. 方法论：STEP-HRL 框架
STEP-HRL 的核心思想是通过**分层抽象**和**进度总结**，将长程历史压缩为增强的单步状态转移。
*   **三层策略架构**：
    1.  **高层策略 ($\pi_h$)**：负责生成子任务（Subtasks）。它以任务指令、已完成子任务序列（全局进度）和前一子任务的最终局部进度为输入。
    2.  **局部进度策略 ($\pi_p$)**：核心创新点。它是一个迭代模块，负责将子任务内的交互细节（当前动作、观测、前一进度）总结为一段紧凑的文字，保留关键信息并控制字数。
    3.  **底层策略 ($\pi_l$)**：负责执行具体动作。它仅以当前子任务、当前观测和局部进度为输入，实现“步级”决策。
*   **训练流程**：
    1.  **行为克隆（BC）初始化**：利用专家数据（通过 DeepSeek 标注子任务和进度）对三个策略进行联合微调。
    2.  **步级离线强化学习（Offline RL）**：采用隐式 Q 学习（IQL）范式。通过值函数 $V$ 和动作值函数 $Q$ 的期望回归，在不偏离行为策略的前提下优化策略。三个策略共享骨干参数，但拥有独立的 Critic 网络。

*   **架构创新**：引入“局部进度策略”作为状态压缩器，巧妙解决了 LLM 智能体长上下文依赖的痛点。
*   **步级决策**：将决策过程从“轨迹预测”回归到经典的“状态-动作”映射，更符合强化学习的本质。
*   **参数高效**：三个策略共享骨干参数，既减少了显存占用，又促进了高低层之间的知识共享。
