## 摘要
多智能体大语言模型（LLM）编排系统面临上下文污染问题：当 N 个并发智能体竞争编排器的上下文窗口时，每个智能体的任务状态、部分输出和待处理问题会干扰其他所有智能体的引导交互，从而降低决策质量。我们引入了动态注意力上下文范围界定（DACS），这是一种使编排器以两种非对称模式运行的机制。在注册（Registry）模式下，它仅保留轻量级的单智能体状态摘要（每个不超过 200 个 token），保持对所有智能体和用户的响应。当某个智能体发出引导请求（SteeringRequest）时，编排器进入聚焦（Focus(a_i)）模式，注入该智能体 a_i 的完整上下文，同时将所有其他智能体压缩为其注册条目。这种上下文隔离是由智能体触发的、非对称的且确定性的：在引导过程中，上下文窗口精确包含 F(a_i) + R_{-i}，在无需上下文压缩或检索的情况下消除了跨智能体污染。我们通过四个实验阶段共 200 次试验对 DACS 进行了评估：第一阶段测试 N 为 {3, 5, 10}（60 次试验）；第二阶段测试智能体异构性和对抗性依赖（60 次试验）；第三阶段测试高达 D=15 的决策密度（40 次试验）；第四阶段使用自主 LLM 智能体处理自由形式的问题（40 次试验，使用 Claude Haiku 4.5）。在所有 8 个合成场景中，DACS 实现了 90.0%–98.4% 的引导准确率，而扁平上下文基准仅为 21.0%–60.0%（全程 p < 0.0001），错误智能体污染率从 28%–57% 下降到 0%–14%，上下文效率比高达 3.53 倍。准确率优势随 N 和 D 的增加而扩大；关键词匹配通过 LLM 作为裁判（LLM-as-judge）在所有阶段进行了验证（平均 kappa=0.909）。在第四阶段，DACS 在 N=3 时优于扁平上下文基准 17.2 个百分点（p=0.0023），在 N=5 时优于 20.4 个百分点（p=0.0008），且优势随 N 增加的趋势得到了两位独立裁判的确认。

## Abstract
Multi-agent LLM orchestration systems suffer from context pollution: when N concurrent agents compete for the orchestrator's context window, each agent's task state, partial outputs, and pending questions contaminate the steering interactions of every other agent, degrading decision quality. We introduce Dynamic Attentional Context Scoping (DACS), a mechanism in which the orchestrator operates in two asymmetric modes. In Registry mode it holds only lightweight per-agent status summaries (<=200 tokens each), remaining responsive to all agents and the user. When an agent emits a SteeringRequest, the orchestrator enters Focus(a_i) mode, injecting the full context of agent a_i while compressing all other agents to their registry entries. Context isolation is agent-triggered, asymmetric, and deterministic: the context window contains exactly F(a_i) + R_{-i} during steering, eliminating cross-agent contamination without requiring context compression or retrieval. We evaluate DACS across four experimental phases totalling 200 trials: Phase 1 tests N in {3,5,10} (60 trials); Phase 2 tests agent heterogeneity and adversarial dependencies (60 trials); Phase 3 tests decision density up to D=15 (40 trials); Phase 4 uses autonomous LLM agents for free-form questions (40 trials, Claude Haiku 4.5). Across all 8 synthetic scenarios, DACS achieves 90.0--98.4% steering accuracy versus 21.0--60.0% for a flat-context baseline (p < 0.0001 throughout), with wrong-agent contamination falling from 28--57% to 0--14% and context efficiency ratios of up to 3.53x. The accuracy advantage grows with N and D; keyword matching is validated by LLM-as-judge across all phases (mean kappa=0.909). DACS outperforms the flat-context baseline by +17.2pp at N=3 (p=0.0023) and +20.4pp at N=5 (p=0.0008) in Phase 4, with the advantage growing with N confirmed by two independent judges.

### 论文总结：Dynamic Attentional Context Scoping (DACS)

#### 1. 核心问题与整体含义（研究动机和背景）
在多智能体大语言模型（LLM）编排系统中，通常由一个中心编排器（Orchestrator）管理多个并发运行的子智能体。当多个智能体同时运行并向编排器请求决策引导（Steering）时，编排器的上下文窗口会充斥着所有智能体的任务状态、中间输出和待处理问题。这种现象被称为**上下文污染（Context Pollution）**。
*   **痛点**：无关智能体的上下文会干扰编排器对当前目标智能体的决策判断，导致决策准确率随智能体数量增加而急剧下降。
*   **目标**：实现一种动态的、隔离的上下文管理机制，确保编排器在引导特定智能体时，只关注该智能体的相关信息。

#### 2. 论文提出的方法论：DACS 机制
DACS 提出了一种基于智能体触发的非对称上下文隔离机制，核心是编排器的**状态机切换**：
*   **REGISTRY（注册）模式**：编排器仅持有所有智能体的轻量级摘要（每个 $\le 200$ tokens），用于监控状态和响应用户。
*   **FOCUS ($a_i$)（聚焦）模式**：当智能体 $a_i$ 发出引导请求时，编排器进入此模式。上下文窗口被重新构建为：**$a_i$ 的完整上下文 + 其他所有智能体的压缩摘要**。
*   **关键技术细节**：
    *   **智能体触发**：隔离不是预设的，而是由智能体在遇到无法自主解决的决策点时主动触发。
    *   **非对称性**：被引导者获得全量信息，其他参与者被高度压缩。
    *   **确定性构建**：不依赖复杂的检索（RAG）或概率压缩，而是根据预设规则硬性构建上下文。
    *   **中断协议**：支持高优先级智能体抢占当前的聚焦会话。

#### 3. 实验设计
实验分为四个阶段，共 200 次试验：
*   **场景设置**：涵盖代码编写、研究调研、数据处理、算法实现等 8 个合成场景。
*   **对比基准（Baseline）**：**Flat-Context（扁平上下文）**，即所有并发智能体的全量上下文同时堆叠在编排器的窗口中。
*   **实验阶段**：
    *   **Phase 1**：智能体数量扩展性测试（$N \in \{3, 5, 10\}$）。
    *   **Phase 2**：智能体异构性与对抗性依赖测试（同领域、跨领域、流水线依赖）。

#### 6. 主要结论与发现
*   **准确率大幅提升**：DACS 在所有场景下均显著优于基准线，准确率保持在 90.0%–98.4%，而基准线在 $N=10$ 时跌至 21.0%。
*   **消除污染**：跨智能体关键词污染率从基准线的 28%–57% 降至 DACS 的 0%–14%。

#### 7. 优点
*   **机制简洁高效**：通过简单的状态机和确定性上下文构建解决了复杂的干扰问题，无需复杂的向量检索。
*   **可扩展性强**：上下文增长与智能体数量呈亚线性关系，适合大规模多智能体协作。
*   **验证严谨**：从脚本化 stub 到自主 LLM 智能体的过渡验证，增强了结论的生态有效性。
