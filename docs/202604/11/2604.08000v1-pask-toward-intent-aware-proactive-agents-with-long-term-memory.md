## 摘要
主动性是通用人工智能（AGI）的核心期望。先前的研究很大程度上局限于实验室环境，在现实世界的主动式智能体方面存在明显差距，涉及深度、复杂性、歧义性、精确度以及实时性约束。我们研究了这一场景，其中有效的干预需要从持续的上下文中推断潜在需求，并在延迟和长程约束下，将行动建立在不断演进的用户记忆之上。我们首先提出了 DD-MM-PAS（需求检测、记忆建模、主动式智能体系统），作为流式主动 AI 智能体的通用范式。我们在 Pask 中实例化了这一范式，包括用于需求检测（DD）的流式 IntentFlow 模型、用于长期记忆建模（MM）的混合记忆（工作区、用户、全局）以及 PAS 基础设施框架，并介绍了这些组件如何形成闭环。我们还推出了 LatentNeeds-Bench，这是一个基于用户授权数据构建并经过数千轮人工编辑精炼而成的真实世界基准测试。实验表明，在延迟约束下，IntentFlow 的表现与领先的 Gemini3-Flash 模型相当，同时能够识别更深层次的用户意图。

## Abstract
Proactivity is a core expectation for AGI. Prior work remains largely confined to laboratory settings, leaving a clear gap in real-world proactive agent: depth, complexity, ambiguity, precision and real-time constraints. We study this setting, where useful intervention requires inferring latent needs from ongoing context and grounding actions in evolving user memory under latency and long-horizon constraints. We first propose DD-MM-PAS (Demand Detection, Memory Modeling, Proactive Agent System) as a general paradigm for streaming proactive AI agent. We instantiate this paradigm in Pask, with streaming IntentFlow model for DD, a hybrid memory (workspace, user, global) for long-term MM, PAS infra framework and introduce how these components form a closed loop. We also introduce LatentNeeds-Bench, a real-world benchmark built from user-consented data and refined through thousands of rounds of human editing. Experiments show that IntentFlow matches leading Gemini3-Flash models under latency constraints, while identifying deeper user intent.

### 2. 方法论：DD-MM-PAS 范式

*   **DD (Demand Detection) - IntentFlow 模型**：
    *   **核心思想**：将主动干预建模为流式决策过程。模型每步输出一个控制 Token：`<silent>`（保持沉默）、`<fast_intervention>`（基于当前上下文快速响应）、`<full_assistance>`（调用记忆和工具进行深度支持）。
    *   **双模型架构**：由 **Demand Detector**（主模型，负责决策和生成）和 **MemLoader**（辅助模型，负责从检索到的记忆中提炼关键证据）组成。

*   **MM (Memory Modeling) - 自演进层次化记忆**：
    *   **用户记忆 (User Memory)**：类似缓存，存储稳定的个人画像和偏好。
    *   **工作区记忆 (Workspace Memory)**：类似内存，维护当前会话的动态状态。
    *   **全局记忆 (Global Memory)**：类似外存，采用树状结构存储长期历史，通过异步 RAG 进行检索，支持“懒合并”和“有界缩放”以防止计算爆炸。

*   **PAS (Proactive Agent System) - 系统实现**：
    *   构建了一个始终在线（Always-on）的多进程运行环境，协调前端感知（眼镜、手机）、后端调度和 AI 模型池（包含 20 多个专家模型）。

### 6. 主要结论与发现

*   **系统性强**：不只是一个模型，而是一套从感知到记忆再到执行的完整闭环系统架构。
*   **实时性优化**：通过流式决策 Token 和异步记忆检索，解决了主动式 AI 常见的响应延迟问题。
*   **高质量数据**：LatentNeeds-Bench 来源于真实世界且经过大量人工精修，填补了主动式 AI 缺乏标准评测集的空白。
