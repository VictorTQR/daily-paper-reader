## 摘要
智能体技能是程序性知识和可执行资源的结构化包，智能体在推理时动态加载这些技能，这已成为增强大语言模型（LLM）智能体的一种可靠机制。然而，推理时的技能增强存在根本性的局限：检索噪声会引入无关的引导，注入的技能内容会带来巨大的 Token 开销，且模型从未真正习得它仅仅是在遵循的知识。我们探究技能是否可以被内化到模型参数中，从而在无需任何运行时技能检索的情况下实现零样本自主行为。我们提出了 SKILL0，这是一个专为技能内化设计的上下文内强化学习框架。SKILL0 引入了一种训练阶段的课程学习方案，从完整的技能上下文开始并逐步将其撤出。技能在离线状态下按类别分组，并与交互历史一起渲染成紧凑的视觉上下文，从而教导模型进行工具调用和多轮任务完成。随后，动态课程会评估每个技能文件的同策略（on-policy）帮助程度，在随线性衰减的预算内，仅保留当前策略仍能从中获益的技能，直到智能体在完全零样本的设置下运行。广泛的智能体实验表明，SKILL0 相比标准强化学习基线取得了显著提升（ALFWorld 提升 9.7%，Search-QA 提升 6.6%），同时保持了每步少于 0.5k Token 的高效上下文。我们的代码可在 https://github.com/ZJU-REAL/SkillZero 获取。

## Abstract
Agent skills, structured packages of procedural knowledge and executable resources that agents dynamically load at inference time, have become a reliable mechanism for augmenting LLM agents. Yet inference-time skill augmentation is fundamentally limited: retrieval noise introduces irrelevant guidance, injected skill content imposes substantial token overhead, and the model never truly acquires the knowledge it merely follows. We ask whether skills can instead be internalized into model parameters, enabling zero-shot autonomous behavior without any runtime skill retrieval. We introduce SKILL0, an in-context reinforcement learning framework designed for skill internalization. SKILL0 introduces a training-time curriculum that begins with full skill context and progressively withdraws it. Skills are grouped offline by category and rendered with interaction history into a compact visual context, teaching he model tool invocation and multi-turn task completion. A Dynamic Curriculum then evaluates each skill file's on-policy helpfulness, retaining only those from which the current policy still benefits within a linearly decaying budget, until the agent operates in a fully zero-shot setting. Extensive agentic experiments demonstrate that SKILL0 achieves substantial improvements over the standard RL baseline (+9.7\% for ALFWorld and +6.6\% for Search-QA), while maintaining a highly efficient context of fewer than 0.5k tokens per step. Our code is available at https://github.com/ZJU-REAL/SkillZero.

# SKILL0: 用于技能内化的上下文内智能体强化学习论文总结

### 1. 核心问题与研究动机
*   **核心问题**：当前的 LLM 智能体主要依赖“推理时技能增强”（Inference-time Skill Augmentation），即通过检索技能库并将其作为 Prompt 注入模型。这种方式存在三大缺陷：
    1.  **检索噪声**：无关技能会干扰上下文。
    2.  **Token 开销**：多轮对话中注入技能描述会导致 Token 成本激增。
    3.  **缺乏真实习得**：模型只是在“遵循”指令，而非真正“掌握”知识，能力留在 Context 中而非参数中。
*   **研究动机**：探索是否能通过强化学习（RL）将外部技能“内化”到模型参数中，实现“训练时有技能辅助，推理时零技能检索”的自主智能体。

### 2. 方法论
SKILL0 提出了**上下文内强化学习（In-Context Reinforcement Learning, ICRL）**框架，核心流程如下：
*   **视觉上下文渲染（Context Rendering）**：为了降低 Token 开销，将交互历史和技能文本渲染为紧凑的图像，利用多模态模型的视觉编码器进行处理，并允许模型自适应生成压缩比。
*   **动态课程学习（Dynamic Curriculum）**：
    *   **线性预算衰减**：设置技能预算 $M(s)$，随训练阶段线性减少，迫使模型减少对外部 Prompt 的依赖。
    *   **帮助度驱动筛选（Helpfulness-Driven）**：在训练过程中，通过对比“有技能”和“无技能”下的验证表现，计算每个技能文件的同策略帮助度 $\Delta k$。
    *   **过滤与排序**：仅保留 $\Delta k > 0$ 的技能，并按帮助度降序排列，优先保留对当前模型提升最大的技能。
*   **优化目标**：采用复合奖励函数，同时优化任务成功率和 Token 压缩效率，通过 RL 算法（如 GRPO 的变体）更新模型参数。

### 3. 实验设计
*   **数据集/场景**：
    *   **ALFWorld**：具身智能环境，包含 6 类家庭任务（如清洁、加热、放置物品等）。
    *   **Search-based QA**：搜索增强问答，涵盖单跳（NQ, TriviaQA）和多跳（HotpotQA, Bamboogle 等）共 7 个数据集。
*   **Benchmark 与对比方法**：
    *   **基线模型**：Qwen2.5-VL (3B 和 7B 版本)。
    *   **对比方法**：Zero-Shot、Few-Shot（带技能 Prompt）、标准 RL（GRPO）、视觉压缩 RL（AgentOCR）、技能增强 RL（SkillRL）、自进化框架（EvolveR）以及 GPT-4o 等闭源模型。

### 4. 资源与算力
*   **硬件环境**：使用了 **4 张 NVIDIA H800 GPU**。
*   **训练细节**：
    *   训练步数：最多 180 步。
    *   ALFWorld：每批次 16 个任务，每个 Prompt 进行 8 次采样（Rollouts）。
    *   Search-QA：每批次 128 个任务。
    *   最大 Prompt 长度：3072 到 4096 Token。

### 5. 实验数量与充分性
*   **实验规模**：在两个完全不同的领域（具身导航与知识搜索）进行了大规模验证，涵盖了从 3B 到 7B 的模型参数梯度。
*   **消融实验**：非常充分。作者对比了不同的技能预算方案（如固定全量、固定少量、无技能训练）、不同的动态筛选策略（无过滤、无排序）以及验证间隔 $d$ 的影响。
*   **客观性**：实验采用了严格的“推理时无技能”对比协议，确保性能提升确实来自于参数内化而非推理时的 Prompt 技巧。

### 6. 主要结论与发现
*   **性能显著提升**：在零样本推理设置下，SKILL0 在 ALFWorld 上比 AgentOCR 提升了 **9.7%**，在 Search-QA 上提升了 **6.6%**。
*   **Token 效率极高**：推理时每步消耗的 Token 少于 **0.5k**，比传统的技能增强方法（如 SkillRL）降低了 5 倍以上的成本。
*   **内化规律**：实验发现技能的“帮助度”随训练呈现“先升后降”的曲线，证明了技能在训练早期作为“脚手架”辅助探索，后期则成功转化为模型内在能力。

### 7. 优点
*   **范式突破**：首次将“技能内化”作为显式的强化学习训练目标，解决了智能体对长 Prompt 的依赖问题。
*   **动态自适应**：通过帮助度指标动态调整训练内容，比固定的课程学习更具鲁棒性。
*   **端到端优化**：结合视觉压缩技术，在提升性能的同时极大地优化了推理延迟和成本。

### 8. 不足与局限
*   **初始依赖性**：内化效果受限于初始技能库（SkillBank）的质量，如果初始技能本身有误，模型可能会内化错误的策略。
*   **离线分组开销**：技能与验证子任务的关联需要预先进行“相关性驱动的分组”，这在扩展到全新、未知的领域时可能需要额外的人工标注或预处理。
*   **视觉编码瓶颈**：虽然视觉渲染节省了 Token，但对于极度精细的文本信息，视觉编码器可能存在感知精度上限。
