## 摘要
重排序器在优化检索增强生成（RAG）的检索结果中起着至关重要的作用。然而，目前的重排序模型通常是在静态的人工标注相关性标签上进行孤立优化的，与下游生成过程脱节。这种孤立导致了根本性的不匹配：被信息检索指标认定为主题相关的文档，往往无法提供大语言模型（LLM）生成精确答案所需的实际效用。为了弥补这一差距，我们引入了重排序偏好优化（RRPO），这是一种强化学习框架，直接将重排序与 LLM 的生成质量对齐。通过将重排序建模为序列决策过程，RRPO 利用 LLM 反馈来优化上下文效用，从而消除了对昂贵人工标注的需求。为了确保训练稳定性，我们进一步引入了一种参考锚定的确定性基准。在知识密集型基准测试上的广泛实验表明，RRPO 显著优于强基线模型，包括强大的 list-wise 重排序器 RankZephyr。进一步的分析强调了该框架的通用性：它可以无缝泛化到不同的阅读器（如 GPT-4o），与 Query2Doc 等查询扩展模块正交集成，并且在面对带有噪声的监督者训练时依然保持鲁棒性。

## Abstract
Rerankers play a pivotal role in refining retrieval results for Retrieval-Augmented Generation. However, current reranking models are typically optimized on static human annotated relevance labels in isolation, decoupled from the downstream generation process. This isolation leads to a fundamental misalignment: documents identified as topically relevant by information retrieval metrics often fail to provide the actual utility required by the LLM for precise answer generation. To bridge this gap, we introduce ReRanking Preference Optimization (RRPO), a reinforcement learning framework that directly aligns reranking with the LLM's generation quality. By formulating reranking as a sequential decision-making process, RRPO optimizes for context utility using LLM feedback, thereby eliminating the need for expensive human annotations. To ensure training stability, we further introduce a reference-anchored deterministic baseline. Extensive experiments on knowledge-intensive benchmarks demonstrate that RRPO significantly outperforms strong baselines, including the powerful list-wise reranker RankZephyr. Further analysis highlights the versatility of our framework: it generalizes seamlessly to diverse readers (e.g., GPT-4o), integrates orthogonally with query expansion modules like Query2Doc, and remains robust even when trained with noisy supervisors.

### 1. 核心问题与研究背景
在检索增强生成（RAG）系统中，重排序器（Reranker）负责从初筛文档中挑选最相关的部分。然而，传统重排序器通常基于静态的人工标注（如相关性得分）进行独立优化，这导致了**检索与生成的脱节**：
*   **相关性不等于效用**：被检索指标（如 NDCG）认为“相关”的文档，未必能帮助大语言模型（LLM）生成准确答案。
*   **孤立优化**：重排序器不了解下游 LLM 的偏好，无法根据生成质量进行动态调整。
*   **标注成本高**：高质量的人工相关性标注昂贵且难以覆盖所有领域。

### 2. 方法论：重排序偏好优化（RRPO）
论文提出了 **RRPO** 框架，其核心思想是将重排序过程建模为一个**序列决策过程（MDP）**，并利用强化学习（RL）直接优化 LLM 的生成质量。

*   **MDP 建模**：
    *   **状态 (State)**：当前尚未被选中的候选文档集合。
    *   **动作 (Action)**：从剩余集合中选择一个文档。
    *   **奖励 (Reward)**：将选定的文档序列输入 LLM Reader，根据生成结果与标准答案的匹配度（EM、F1、Hit）计算奖励。
*   **关键技术：参考锚定确定性基准 (Reference-anchored Deterministic Baseline)**：
    *   **挑战**：在 RAG 场景中，LLM 的反馈奖励是离散且高方差的，传统的 PPO 算法中训练一个连续的 Critic 网络（价值函数）非常困难。
    *   **解决方案**：放弃训练 Critic 网络，转而使用一个固定的参考策略（Reference Policy，如未优化的原始重排序器）进行贪婪搜索，将其得到的奖励作为基准值（Baseline）。只有当当前策略的表现优于参考策略时，才会产生正向优势（Advantage），从而稳定训练。
*   **优化目标**：采用 PPO-clip 机制结合 KL 散度惩罚，确保策略更新平滑，防止模型偏离原始分布过远。

*   **端到端对齐**：解决了检索与生成目标不一致的根本问题。
*   **无需人工标注**：完全依赖 LLM 反馈，实现了自动化的闭环优化。
*   **训练稳定性**：通过“参考锚定基准”巧妙避开了 RLHF 中 Critic 网络难以收敛的坑。

*   **召回率瓶颈**：重排序器只能处理初筛后的文档。如果初始检索（如 BM25）未能召回包含答案的文档，RRPO 优化得再好也无法挽回。

*   **任务依赖性**：奖励函数（EM/F1）主要针对简短回答的问答任务，对于长文本生成或摘要任务的适用性尚待验证。
