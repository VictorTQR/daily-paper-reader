## 摘要
大型基础模型为自主系统提供了强大的推理能力，但将语义意图映射到可靠的实时控制仍然具有挑战性。现有方法要么 (i) 让大语言模型 (LLM) 直接生成轨迹——这种方式脆弱、难以验证且容易产生延迟；要么 (ii) 在线调整模型预测控制 (MPC) 的目标——将缓慢的深思熟虑与快速控制混合在一起，模糊了接口。我们提出了智能体快慢规划 (Agentic Fast-Slow Planning)，这是一个在自然时间尺度上解耦感知、推理、规划和控制的分层框架。该框架包含两个桥梁：Perception2Decision 使用车载视觉语言模型 (VLM) 检测器将场景压缩为以自我为中心的拓扑结构，然后通过云端 LLM 决策器将其映射为符号化驾驶指令，在保持可解释性的同时减少了带宽和延迟；Decision2Trajectory 将指令转换为可执行路径，其中语义引导的 A* (Semantic-Guided A*) 将源自语言的软成本嵌入到经典搜索中，使解偏向于可行轨迹，而智能体细化模块 (Agentic Refinement Module) 则利用反馈和记忆来调整规划器的超参数。最后，MPC 实时跟踪轨迹，并针对困难情况提供可选的云端引导参考。在 CARLA 中的实验表明，与纯 MPC 和 A* 引导的 MPC 基准相比，智能体快慢规划提高了扰动下的鲁棒性，横向偏差减少了高达 45%，完成时间缩短了 12% 以上。代码可在 https://github.com/cjychenjiayi/icra2026_AFSP 获取。

## Abstract
Large foundation models enable powerful reasoning for autonomous systems, but mapping semantic intent to reliable real-time control remains challenging. Existing approaches either (i) let Large Language Models (LLMs) generate trajectories directly - brittle, hard to verify, and latency-prone - or (ii) adjust Model Predictive Control (MPC) objectives online - mixing slow deliberation with fast control and blurring interfaces. We propose Agentic Fast-Slow Planning, a hierarchical framework that decouples perception, reasoning, planning, and control across natural timescales. The framework contains two bridges. Perception2Decision compresses scenes into ego-centric topologies using an on-vehicle Vision-Language Model (VLM) detector, then maps them to symbolic driving directives in the cloud with an LLM decision maker - reducing bandwidth and delay while preserving interpretability. Decision2Trajectory converts directives into executable paths: Semantic-Guided A* embeds language-derived soft costs into classical search to bias solutions toward feasible trajectories, while an Agentic Refinement Module adapts planner hyperparameters using feedback and memory. Finally, MPC tracks the trajectories in real time, with optional cloud-guided references for difficult cases. Experiments in CARLA show that Agentic Fast-Slow Planning improves robustness under perturbations, reducing lateral deviation by up to 45% and completion time by over 12% compared to pure MPC and an A*-guided MPC baseline. Code is available at https://github.com/cjychenjiayi/icra2026_AFSP.

### 2. 论文提出的方法论
AFSP 框架由三个层次（推理、规划、控制）和两个关键桥梁组成：

*   **Perception2Decision（感知到决策桥梁）**：
    *   **车载端（边缘）**：使用轻量化 VLM（如 InternVL-2B）作为拓扑检测器，将原始图像压缩为以自我为中心的**拓扑图（Topology Graph）**，包含物体类别、距离、方位等关键语义，极大降低了上传云端的带宽需求。
    *   **云端**：LLM 决策器基于拓扑图进行思维链（CoT）推理，输出符号化的驾驶指令序列（如 `LEFT`, `KEEP`, `RIGHT`）和驾驶风格。
*   **Decision2Trajectory（决策到轨迹桥梁）**：
    *   **语义引导 A* (Semantic-Guided A*)**：在经典 A* 算法中引入“软语义成本”。算法根据 LLM 指令与当前搜索动作的匹配程度（正确、延迟、错误、过度）分配不同的代价值，使搜索结果在满足几何可行性的同时，倾向于符合语义意图。
    *   **智能体细化模块 (Agentic Refinement Module)**：利用 LLM 作为一个闭环调节器，根据规划器的反馈（如是否碰撞、是否偏离指令）自动调整 A* 的超参数（如各项成本权重），模拟人类调参过程。
*   **实时控制层**：
    *   **云端引导 MPC**：使用模型预测控制实时跟踪 A* 生成的参考轨迹。在复杂场景下，MPC 会优先参考云端生成的全局路径，以保证全局安全性。

### 6. 论文的主要结论与发现
*   **性能提升**：AFSP 在所有场景下均表现最优。相比纯 MPC，其最大横向偏差降低了约 **45%**，完成时间缩短了 **12%** 以上。
*   **鲁棒性**：在地图发生平移扰动（模拟感知/定位误差）时，语义引导的 A* 能够纠正经典 A* 的偏差，保持意图一致性。
*   **效率**：通过拓扑图压缩，云端推理的延迟显著低于直接处理图像的 VLM，且决策质量相当。
*   **自适应性**：智能体细化模块能有效自动化超参数调节，无需人工干预即可适配不同场景。

### 7. 优点（亮点）
*   **架构解耦**：成功实现了“慢思考（云端语义）”与“快行动（边缘控制）”的协同，解决了大模型上车的实时性难题。
*   **软约束机制**：将 LLM 指令作为 A* 的“软成本”而非“硬约束”，既尊重了大模型的意图，又保留了经典算法处理几何冲突的底线。
*   **闭环调参**：引入 Agent 自动调参机制，提升了经典规划算法在长尾场景下的适应力。

### 8. 不足与局限

*   **计算开销**：虽然感知端轻量化了，但云端运行 DeepSeek-V3 等超大模型仍需昂贵的计算资源支持。
