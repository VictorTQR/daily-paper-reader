## 摘要
强化学习（RL）已被证明能显著提高小型和大型语言模型（LLM）的推理能力，但现有方法通常依赖于可验证的奖励，因此需要标准答案（ground truth）标签。我们提出了一种强化学习框架，该框架利用作为裁判的 LLM 对大量无标签数据上的模型输出进行评估并提供奖励，从而实现无标签知识蒸馏，并取代了对标准答案监督的需求。值得注意的是，该裁判通过单 token 输出进行操作，使得奖励计算非常高效。当与可验证奖励相结合时，我们的方法在数学推理基准测试中取得了显著的性能提升。这些结果表明，基于 LLM 的评估器可以为强化学习微调产生有效的训练信号。

## Abstract
Reinforcement Learning (RL) has been shown to substantially improve the reasoning capability of small and large language models (LLMs), but existing approaches typically rely on verifiable rewards, hence ground truth labels. We propose an RL framework that uses rewards from an LLM that acts as a judge evaluating model outputs over large amounts of unlabeled data, enabling label-free knowledge distillation and replacing the need of ground truth supervision. Notably, the judge operates with a single-token output, making reward computation efficient. When combined with verifiable rewards, our approach yields substantial performance gains across math reasoning benchmarks. These results suggest that LLM-based evaluators can produce effective training signals for RL fine-tuning.

### 2. 方法论
*   **核心思想**：将 LLM 裁判从静态评估器转变为动态奖励函数。学生模型生成推理轨迹（如 Python 代码），教师模型直接对该轨迹打分，分数作为 RL 的奖励信号。
*   **关键技术细节**：
    *   **单 Token 奖励计算**：为了提高效率，裁判 LLM 被限定仅输出“Yes”或“No”。通过提取这两个 token 的 Logits（未归一化概率），计算连续的标量奖励：$s = \sigma(\frac{\log p(Yes) - \log p(No)}{\tau})$。这种方法避免了生成长文本，极大地加快了训练速度。
    *   **多源奖励融合**：最终目标函数是三种损失的加权组合：
        1.  **$L_{VR}$（可验证奖励）**：基于有标签数据的答案对错。
        2.  **$L_{YoN}$（裁判奖励）**：基于无标签数据，由教师 LLM（如 Qwen3-8B）判定。
        3.  **$L_{Rerank}$（重排序奖励）**：利用预训练的分类器模型提供额外的质量信号。
    *   **算法流程**：采用 PPO（近端策略优化）作为主要算法，并对比了 GRPO（组相对策略优化）。学生模型首先通过 SFT 进行热身，随后在混合了有标签和无标签的数据集上进行 RL 训练。

### 6. 主要结论与发现
*   **性能飞跃**：在 125M 和 350M 模型上，该方法比仅用可验证奖励的方法提升了 **5-10 个百分点**。
*   **无标签数据的威力**：在无标签数据上使用“噪声”奖励（LLM 裁判）的效果，甚至可以超过在有限标签数据上使用“精确”奖励的效果。
*   **规模可扩展性**：该方法在 6.7B 模型上依然有效（提升约 1.5 分），证明了其在大模型上的潜力。

### 7. 优点
*   **高效性**：创新的“单 Token Logits”奖励机制，解决了 LLM 作为裁判时的推理延迟瓶颈。
*   **标签无关性**：极大地扩展了 RL 训练的数据来源，使利用互联网规模的无标签数据进行推理训练成为可能。
*   **主动探索**：相比于传统的离线蒸馏，RL 框架允许学生模型在教师的指导下进行自我博弈和空间探索。

### 8. 不足与局限
*   **奖励作弊（Reward Hacking）**：论文坦诚在完全去掉可验证奖励（$\lambda=0$）时，模型容易出现“奖励作弊”现象（例如生成极短或无意义的代码却骗取高分）。
*   **裁判依赖性**：蒸馏效果上限受限于裁判 LLM 的水平。如果裁判模型本身对某些复杂逻辑判断失误，会误导学生模型。
*   **任务局限**：目前主要集中在数学推理（PoT 形式），在更广泛的自然语言任务或其他主观性任务上的表现尚待验证。
