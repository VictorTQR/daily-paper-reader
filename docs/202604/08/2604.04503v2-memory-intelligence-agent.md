## 摘要
深度研究智能体 (DRA) 将大语言模型 (LLM) 的推理能力与外部工具相结合。记忆系统使 DRA 能够利用历史经验，这对于高效推理和自主演化至关重要。现有方法依赖于从记忆中检索相似轨迹来辅助推理，但面临记忆演化效率低下以及存储和检索成本不断增加的关键局限。为了解决这些问题，我们提出了一种新型的记忆智能体 (MIA) 框架，该框架由“管理器-规划器-执行器” (Manager-Planner-Executor) 架构组成。记忆管理器是一个非参数化记忆系统，可以存储压缩后的历史搜索轨迹。规划器是一个参数化记忆智能体，可以针对问题生成搜索计划。执行器是另一个智能体，可以在搜索计划的指导下搜索和分析信息。为了构建 MIA 框架，我们首先采用交替强化学习范式来增强规划器与执行器之间的协作。此外，我们使规划器能够在测试时学习过程中持续演化，在不中断推理过程的情况下，随推理同步进行实时更新。此外，我们在参数化记忆和非参数化记忆之间建立了双向转换循环，以实现高效的记忆演化。最后，我们引入了反思和无监督判断机制，以提升在开放世界中的推理和自我演化能力。在 11 个基准测试上的广泛实验证明了 MIA 的优越性。

## Abstract
Deep research agents (DRAs) integrate LLM reasoning with external tools. Memory systems enable DRAs to leverage historical experiences, which are essential for efficient reasoning and autonomous evolution. Existing methods rely on retrieving similar trajectories from memory to aid reasoning, while suffering from key limitations of ineffective memory evolution and increasing storage and retrieval costs. To address these problems, we propose a novel Memory Intelligence Agent (MIA) framework, consisting of a Manager-Planner-Executor architecture. Memory Manager is a non-parametric memory system that can store compressed historical search trajectories. Planner is a parametric memory agent that can produce search plans for questions. Executor is another agent that can search and analyze information guided by the search plan. To build the MIA framework, we first adopt an alternating reinforcement learning paradigm to enhance cooperation between the Planner and the Executor. Furthermore, we enable the Planner to continuously evolve during test-time learning, with updates performed on-the-fly alongside inference without interrupting the reasoning process. Additionally, we establish a bidirectional conversion loop between parametric and non-parametric memories to achieve efficient memory evolution. Finally, we incorporate a reflection and an unsupervised judgment mechanisms to boost reasoning and self-evolution in the open world. Extensive experiments across eleven benchmarks demonstrate the superiority of MIA.

### 1. 核心问题与整体含义（研究动机和背景）
深度研究智能体（Deep Research Agents, DRAs）通过结合 LLM 推理与外部搜索工具来处理复杂任务。然而，现有的智能体记忆系统存在三大痛点：
*   **效率低下：** 简单的长上下文记忆（Long-context memory）会稀释模型注意力，引入噪声，且随着历史增长，存储和检索成本呈爆炸式增加。
*   **演化困难：** 现有方法多为静态检索，缺乏在推理过程中实时学习和优化策略的能力。
*   **缺乏过程导向：** 传统记忆侧重于“结果是什么”（事实知识），而深度研究更需要“如何获得结果”（搜索路径、失败教训、规划策略）。
**研究动机：** 开发一个能够像人类大脑一样，既能存储具体经验（非参数化），又能将经验内化为本能（参数化），并能在开放世界中自主进化的智能体框架。

### 2. 方法论：核心思想与关键技术
MIA 提出了 **Manager-Planner-Executor（管理者-规划者-执行者）** 架构，核心思想是“解耦存储、规划与执行”：
*   **Memory Manager（非参数化记忆）：** 负责将冗长的搜索轨迹压缩为结构化的“工作流摘要”（Workflow Summaries），并基于语义相似度、价值奖励（成功率）和频率奖励进行混合检索。
*   **Planner（参数化记忆）：** 这是一个可训练的智能体，负责根据问题和检索到的记忆生成搜索计划。它通过内化历史经验，将非参数化记忆转化为模型参数。
*   **Executor（执行终端）：** 负责具体工具调用（ReAct 模式）。它遵循 Planner 的计划，并在遇到困难时向 Planner 反馈。
*   **关键技术细节：**
    *   **交替强化学习（Alternating RL）：** 使用 GRPO 算法，第一阶段训练 Executor 遵循计划，第二阶段训练 Planner 吸收记忆并进行反思（Reflect-Replan）。
    *   **测试时学习（Test-Time Learning, TTL）：** 在推理阶段，Planner 边推理边通过在线 RL 更新参数，实现“即学即用”。
    *   **无监督自演化：** 针对无标注环境，设计了模拟学术评审的“Reviewer-AC”机制，通过逻辑、事实、格式三个维度的专家模型打分，为 RL 提供奖励信号。

*   **训练配置：** Executor 训练 8 个 epoch，Planner 训练 4 个 epoch。TTL 阶段学习率设为 $1 \times 10^{-6}$。

### 5. 实验数量与充分性
*   **实验规模：** 论文进行了大规模的跨模态实验，涵盖了从基础事实问答到复杂多步推理的多种场景。
*   **消融实验：** 非常详尽，分别验证了“仅记忆”、“仅规划”、“反思机制”、“强化学习训练”以及“TTL”对最终性能的贡献。
*   **客观性：** 采用了统一的工具环境和 LLM-as-a-Judge 评分标准，并对比了多种闭源 SOTA 模型，实验设计较为严谨、公平。

### 6. 主要结论与发现
*   **性能飞跃：** MIA 显著提升了现有 SOTA 模型的表现。例如，将 GPT-5.4 的性能在 LiveVQA 上提升了 9%。
*   **小模型逆袭：** 搭载 MIA 框架的 **7B 模型**（Qwen2.5-VL-7B）在平均表现上超过了未搭载该框架的 **32B 模型**，甚至在多个指标上逼近 Gemini-3-Flash。
*   **记忆噪声验证：** 实验证实传统的长上下文记忆（如 RAG, Mem0）在复杂任务中往往因引入噪声而导致性能下降，而 MIA 的压缩工作流机制有效解决了此问题。
*   **持续进化：** 在无监督环境下，MIA 通过多次迭代（Epoch 1-3），性能呈现稳步上升趋势，证明了其自主进化的潜力。

### 7. 优点与亮点
*   **双向记忆循环：** 巧妙地结合了非参数化记忆的“精确性”和参数化记忆的“高效性”。
*   **在线 TTL 机制：** 实现了在测试阶段不中断推理的实时参数更新，这在目前的智能体研究中具有前瞻性。
*   **无监督评审架构：** 提出的 Reviewer-AC 机制为解决智能体在真实世界中缺乏 Ground Truth 的奖励建模问题提供了新思路。

### 8. 不足与局限
*   **计算开销：** 虽然 TTL 提升了性能，但在线进行强化学习更新（Rollout + Backward）会显著增加推理时的计算资源消耗和延迟。
*   **模型依赖：** 框架的有效性高度依赖于作为“判断者”和“管理器”的高阶模型（如 Qwen3-32B），如果基础判断模型存在偏见，可能会导致演化方向偏差。
*   **环境动态性：** 目前主要针对静态知识搜索，对于需要实时交互、环境状态剧烈变化的动态任务（如机器人控制或实时游戏）尚未验证。
