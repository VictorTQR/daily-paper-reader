## 摘要
组相对策略优化（GRPO）广泛用于具有可验证奖励的强化学习，但它经常遭受优势崩溃（advantage collapse）的问题：当一个组中的所有采样序列（rollouts）获得相同的奖励时，该组产生的相对优势为零，从而没有学习信号。例如，如果一个问题对推理器来说太难，所有采样的序列可能都是错误的并获得零奖励。最近的工作通过向此类难题添加提示或辅助支架来解决这一问题，使推理器产生混合结果并恢复非零更新。然而，现有的提示通常是固定的，而不是适应当前的推理器，并且在有提示输入下产生学习信号的提示并不一定能改进测试时使用的无提示策略。为此，我们提出了强化学习中的提示学习（HiLL），这是一个在强化学习期间联合训练提示器策略和推理器策略的框架。对于每个难题，提示器根据当前推理器的错误采样序列在线生成提示，使提示生成能够适应推理器不断演变的错误。我们进一步引入了提示依赖度（hint reliance），用于衡量正确的提示轨迹对提示的依赖程度。我们推导出一个可迁移性结果，表明较低的提示依赖度意味着从有提示成功到无提示成功的更强迁移，并利用这一结果定义了用于训练提示器的迁移加权奖励。因此，HiLL 倾向于那些不仅能恢复信息丰富的 GRPO 组，还能产生更有可能改进原始无提示策略的信号的提示。在多个基准测试上的实验表明，HiLL 一致优于 GRPO 和之前的基于提示的基准，证明了自适应和迁移感知的提示学习在强化学习中的价值。代码可在 https://github.com/Andree-9/HiLL 获取。

## Abstract
Group Relative Policy Optimization (GRPO) is widely used for reinforcement learning with verifiable rewards, but it often suffers from advantage collapse: when all rollouts in a group receive the same reward, the group yields zero relative advantage and thus no learning signal. For example, if a question is too hard for the reasoner, all sampled rollouts can be incorrect and receive zero reward. Recent work addresses this issue by adding hints or auxiliary scaffolds to such hard questions so that the reasoner produces mixed outcomes and recovers a non-zero update. However, existing hints are usually fixed rather than adapted to the current reasoner, and a hint that creates learning signal under the hinted input does not necessarily improve the no-hint policy used at test time. To this end, we propose Hint Learning for Reinforcement Learning (HiLL), a framework that jointly trains a hinter policy and a reasoner policy during RL. For each hard question, the hinter generates hints online conditioned on the current reasoner's incorrect rollout, allowing hint generation to adapt to the reasoner's evolving errors. We further introduce hint reliance, which measures how strongly correct hinted trajectories depend on the hint. We derive a transferability result showing that lower hint reliance implies stronger transfer from hinted success to no-hint success, and we use this result to define a transfer-weighted reward for training the hinter. Therefore, HiLL favors hints that not only recover informative GRPO groups, but also produce signals that are more likely to improve the original no-hint policy. Experiments across multiple benchmarks show that HiLL consistently outperforms GRPO and prior hint-based baselines, demonstrating the value of adaptive and transfer-aware hint learning for RL. The code is available at https://github.com/Andree-9/HiLL.

### 1. 核心问题与整体含义（研究动机和背景）
在具有可验证奖励的强化学习（RLVR）中，**GRPO（组相对策略优化）** 是目前的主流算法。然而，GRPO 存在一个致命的缺陷——**优势坍缩（Advantage Collapse）**：
*   **问题背景**：GRPO 通过对比同一组内多个采样序列（Rollouts）的奖励来计算优势。如果一个问题太难，模型的所有采样结果都是错误的（奖励均为 0），那么组内的相对优势就全部为 0，导致模型无法获得任何学习信号，陷入“无法通过错误学习”的困境。
*   **核心问题**：现有的解决方法（如固定提示或辅助支架）通常是静态的，无法适应模型不断变化的错误模式；且如果提示过于直接（给出了答案），模型学会的是“依赖提示”而非“提升自身推理能力”，导致这些信号无法迁移到测试时的无提示场景。

### 2. 方法论：核心思想与关键技术
HiLL 的核心思想是**联合训练一个“提示器”（Hinter）和一个“推理器”（Reasoner）**，让提示器学会生成既能产生学习信号、又具有高迁移性的提示。

*   **失败驱动的在线提示生成**：当推理器在某个问题上采样全错时，提示器会根据“问题 + 推理器的错误尝试 + 参考答案”在线生成提示。这使得提示能精准针对模型当前的弱点。
*   **提示依赖度（Hint Reliance）**：论文引入了一个关键度量 $\rho$，定义为：模型在“有提示”下生成正确答案的概率与“无提示”下概率的对数差。
    *   **低依赖度**：意味着正确路径在原题下也比较合理，提示只是起到了“点拨”作用，迁移性强。
    *   **高依赖度**：意味着模型完全是靠提示“作弊”才对的，迁移性弱。
*   **迁移加权奖励（Transfer-Weighted Reward）**：提示器的训练目标不仅是让推理器做对（信号创建），还要最小化提示依赖度（信号迁移）。奖励函数结合了这两点，惩罚那些直接给出计算步骤或答案的“剧透型”提示。
*   **联合优化流程**：
    1. 推理器尝试解题，识别全错组。
    2. 提示器生成多个候选提示。
    3. 推理器在提示下重新采样。
    4. 计算迁移加权奖励，通过 GRPO 同时更新提示器和推理器。

### 6. 主要结论与发现
*   **性能领先**：HiLL 在所有数学和通用推理基准上均优于标准 GRPO 和现有的提示基线。
*   **迁移性的重要性**：消融实验证明，如果不考虑“提示依赖度”，提示器会倾向于生成冗长、直接给出计算过程的提示，虽然能产生信号，但对模型自身能力的提升有限。
*   **提示风格的演变**：经过 HiLL 训练的提示器生成的提示更短、更具启发性（如提供战略性建议而非具体公式），这有助于模型内化推理逻辑。

### 7. 优点
*   **自适应性强**：提示器随推理器的进步而进化，始终作用于推理器的“能力边界”。
*   **理论支撑**：推导了迁移边界（Transfer Bound）理论，为提示依赖度的引入提供了数学依据。
*   **通用性**：虽然在数学任务上训练，但在 GPQA 等非数学任务上也表现出良好的泛化能力。

### 8. 不足与局限
*   **计算开销**：在线生成提示和二次采样显著增加了训练时的计算负担（延迟增加 2-3 倍）。
*   **依赖参考答案**：在训练阶段，提示器需要访问参考答案（Ground Truth Solution）来生成有效提示，这限制了其在完全无监督或无答案场景下的应用。
*   **冷启动依赖**：如果底座模型极弱，连提示器都无法引导出任何正确路径，该方法可能失效。
