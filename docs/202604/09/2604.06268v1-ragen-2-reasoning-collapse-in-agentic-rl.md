## 摘要
多轮大语言模型（LLM）智能体的强化学习（RL）训练本质上是不稳定的，且推理质量直接决定了任务性能。熵被广泛用于追踪推理的稳定性。然而，熵仅衡量同一输入内部的多样性，无法判断推理是否真正对不同输入做出了响应。在 RAGEN-2 中，我们发现即使熵保持稳定，模型也可能依赖于看起来多样但与输入无关的固定模板。我们将这种现象称为“模板崩溃”（template collapse），这是一种熵和所有现有指标都无法察觉的失效模式。为了诊断这一失效，我们将推理质量分解为输入内多样性（熵）和跨输入可区分性（互信息，MI），并引入了一系列互信息代理指标用于在线诊断。在各种任务中，互信息与最终性能的相关性远强于熵，使其成为更可靠的推理质量代理指标。我们进一步利用信噪比（SNR）机制解释了模板崩溃。低奖励方差会削弱任务梯度，导致正则化项占据主导地位并抹除跨输入的推理差异。为解决这一问题，我们提出了 SNR 感知过滤（SNR-Aware Filtering），利用奖励方差作为轻量级代理，在每次迭代中筛选高信号提示词。在规划、数学推理、网页导航和代码执行等任务中，该方法一致地提高了输入依赖性和任务性能。

## Abstract
RL training of multi-turn LLM agents is inherently unstable, and reasoning quality directly determines task performance. Entropy is widely used to track reasoning stability. However, entropy only measures diversity within the same input, and cannot tell whether reasoning actually responds to different inputs. In RAGEN-2, we find that even with stable entropy, models can rely on fixed templates that look diverse but are input-agnostic. We call this template collapse, a failure mode invisible to entropy and all existing metrics. To diagnose this failure, we decompose reasoning quality into within-input diversity (Entropy) and cross-input distinguishability (Mutual Information, MI), and introduce a family of mutual information proxies for online diagnosis. Across diverse tasks, mutual information correlates with final performance much more strongly than entropy, making it a more reliable proxy for reasoning quality. We further explain template collapse with a signal-to-noise ratio (SNR) mechanism. Low reward variance weakens task gradients, letting regularization terms dominate and erase cross-input reasoning differences. To address this, we propose SNR-Aware Filtering to select high-signal prompts per iteration using reward variance as a lightweight proxy. Across planning, math reasoning, web navigation, and code execution, the method consistently improves both input dependence and task performance.

#### 1. 核心问题与整体含义（研究动机和背景）
在多轮大语言模型（LLM）智能体的强化学习（RL）训练中，研究者通常使用**奖励（Reward）**监控任务表现，使用**熵（Entropy）**监控推理过程的稳定性。然而，论文发现了一个严重的失效模式：**模板崩溃（Template Collapse）**。
*   **现象**：模型生成的推理链（Reasoning Chains）虽然在单个输入下看起来很丰富（高熵），但对于不同的输入，其推理内容却趋向于雷同的固定模板（低互信息）。
*   **危害**：这种崩溃是“静默”的，现有的熵指标无法检测到。模型虽然能维持格式正确，但推理过程已失去对具体任务目标的敏感性，导致智能体在复杂环境下的可靠性大幅下降。

#### 2. 方法论：核心思想与关键技术
论文提出了一套从诊断到干预的完整框架：
*   **互信息（MI）诊断框架**：将推理质量分解为“输入内多样性”（熵）和“跨输入区分度”（互信息）。
    *   **核心思想**：如果推理是针对输入定制的，那么给定推理链应该能准确找回其对应的输入。
    *   **技术细节**：引入**在簇内交叉评分（In-Batch Cross-Scoring）**。通过计算推理轨迹在当前批次所有提示词（Prompts）下的对数似然，构建评分矩阵，导出“检索准确率”（Retrieval-Acc）和“MI-ZScore”等代理指标，无需额外模型即可在线监测输入依赖性。
*   **信噪比（SNR）机制解释**：
    *   论文发现任务梯度（Task Gradient）的强度受**奖励方差（Reward Variance, RV）**控制。
    *   当 RV 较低时，任务信号微弱，而正则化项（如 KL 散度和熵奖励）产生的梯度是输入无关的“噪声”。此时噪声主导更新，抹除了推理的输入差异。
*   **SNR 感知过滤（SNR-Aware Filtering）**：
    *   **算法流程**：在每次训练迭代中，先对每个 Prompt 进行多次采样（Rollout），计算其奖励方差。
    *   **核心操作**：采用类似核采样（Nucleus Sampling）的 **Top-p 过滤**，仅保留奖励方差总和前 $p$（如 0.9）的 Prompts 进行参数更新，剔除那些无法提供有效学习信号的低方差样本。

#### 6. 主要结论与发现
*   **MI 是更好的监控器**：互信息与任务性能的正相关性（+0.39）远高于熵（-0.14），是诊断推理崩溃的可靠指标。
*   **SNR 决定推理质量**：奖励方差直接控制任务梯度的强度。低方差样本会导致正则化项“反客为主”，诱发模板崩溃。
*   **过滤提升性能**：SNR 感知过滤在几乎所有设置下都提升了成功率（平均提升约 4%-7%），尤其在 Sokoban 等复杂规划任务中提升显著（最高达 +16%）。

#### 7. 优点
*   **视角新颖**：首次识别并定义了智能体 RL 中的“模板崩溃”现象，并从信息论和梯度动力学角度给出了深刻解释。
*   **实用性强**：提出的过滤方法不需要额外的奖励模型或复杂的超参数调节，且能显著降低训练计算开销。
*   **验证全面**：实验覆盖了从 0.5B 小模型到视觉大模型，证明了该失效模式的普遍性和解决方案的通用性。

#### 8. 不足与局限
*   **环境依赖性**：该方法依赖于奖励方差作为信号代理。在奖励极其稀疏或环境噪声极大的极端情况下（如文中提到的 80%-100% 随机性的 FrozenLake），奖励方差可能失效。
*   **探索风险**：过度激进的过滤（$p$ 值过小）可能会导致模型只在容易产生差异的样本上学习，从而缩小探索覆盖面。
*   **单智能体限制**：目前研究仅限于单智能体场景，尚未探讨多智能体强化学习中是否存在类似的推理崩溃传播问题。
