## 摘要
多智能体检索增强生成（RAG）通过为每个智能体分配特定角色，能够支持需要多步操作、多源信息或复杂推理的难题。然而，现有方法依赖于静态的智能体行为和固定的编排策略，导致在多样化的多跳任务中表现欠佳且缺乏鲁棒性。我们识别出两个关键局限：缺乏持续自适应的编排机制，以及单个智能体缺乏行为层面的学习。为此，我们提出了 HERA，这是一个联合演化多智能体编排和特定角色智能体提示词的分层框架。在全局层面，HERA 通过奖励引导的采样和经验积累来优化针对特定查询的智能体拓扑结构。在局部层面，角色感知提示词演化通过信用分配以及沿操作和行为原则的双轴自适应来精炼智能体行为，从而实现有针对性的、基于角色的改进。在六个知识密集型基准测试中，HERA 在保持鲁棒泛化能力和 Token 效率的同时，比最近的基准模型平均提升了 38.69%。拓扑分析揭示了涌现的自组织现象，其中稀疏探索产生了紧凑且高实用性的多智能体网络，展示了高效的协作和鲁棒的推理能力。

## Abstract
Multi-agent Retrieval-Augmented Generation (RAG), wherein each agent takes on a specific role, supports hard queries that require multiple steps and sources, or complex reasoning. Existing approaches, however, rely on static agent behaviors and fixed orchestration strategies, leading to brittle performance on diverse, multi-hop tasks. We identify two key limitations: the lack of continuously adaptive orchestration mechanisms and the absence of behavior-level learning for individual agents. To this end, we propose HERA, a hierarchical framework that jointly evolves multi-agent orchestration and role-specific agent prompts. At the global level, HERA optimizes query-specific agent topologies through reward-guided sampling and experience accumulation. At the local level, Role-Aware Prompt Evolution refines agent behaviors via credit assignment and dual-axes adaptation along operational and behavioral principles, enabling targeted, role-conditioned improvements. On six knowledge-intensive benchmarks, HERA achieves an average improvement of 38.69\% over recent baselines while maintaining robust generalization and token efficiency. Topological analyses reveal emergent self-organization, where sparse exploration yields compact, high-utility multi-agent networks, demonstrating both efficient coordination and robust reasoning.

#### 1. 核心问题与整体含义
论文针对当前**多智能体检索增强生成（Multi-agent RAG）**系统存在的两大核心局限进行了研究：
*   **静态编排（Static Orchestration）：** 现有的系统通常采用固定的流水线或简单的顺序逻辑，无法根据查询（Query）的复杂程度动态调整智能体之间的协作结构（拓扑）。
*   **行为僵化（Brittle Agent Behaviors）：** 单个智能体依赖固定提示词（Prompts），缺乏针对特定任务失败的自我进化能力，且全局监督信号（如最终答案对错）难以精准定位并修复中间环节的错误。
**研究动机：** 提出一种无需更新模型参数、通过“经验”驱动的演化框架，使系统能够像指南针一样，利用过往的成功与失败案例来优化全局协作策略和局部智能体行为。

#### 2. 方法论：HERA 框架
HERA（Hierarchical Evolution RAG）采用三层分层架构，核心思想是将策略优化从“参数更新”转向“上下文经验适配”：

*   **全局编排层（Orchestrator）：** 
    *   受 GRPO（组相对策略优化）启发，但将其提升至**拓扑级别**。编排器针对一个查询采样多组不同的智能体协作拓扑（顺序、并行、依赖关系），通过比较不同拓扑的奖励（准确率与 Token 效率）提取“语义优势”（Semantic Advantages）。
*   **经验库（Experience Library）：** 
    *   作为中间桥梁，存储“概况-见解-效用”（Profile-Insight-Utility）三元组。它通过 ADD（添加）、MERGE（合并）、PRUNE（剪枝）等操作动态维护，为编排器提供针对不同类型查询的先验知识。
*   **局部执行层（Role-aware Prompt Evolution, RoPE）：** 
    *   **角色感知提示演化：** 针对表现不佳的智能体，RoPE 沿两个维度优化其提示词：**操作规则**（针对近期失败的短期修正）和**行为原则**（跨任务的长期策略）。
    *   **拓扑变异（Topology Mutation）：** 当提示词优化无法解决问题时，系统会尝试更换智能体或增加新的验证步骤，实现结构层面的容错。

#### 6. 主要结论与发现
*   **性能卓越：** HERA 在 6 个基准测试中平均比现有最强基准提升了 **38.69%**。
*   **自组织涌现：** 随着经验积累，系统会自动从复杂的、冗余的协作拓扑演化为紧凑、高实用性的网络，Token 消耗在经历初期的探索峰值后会显著下降并趋于平稳。
*   **协同效应：** 消融实验证明，提示词演化对多跳问答至关重要，而经验库则在处理歧义和提高输出精度方面表现突出，两者结合具有非加性的增强效果。

#### 7. 优点
*   **分层演化机制：** 将全局拓扑优化与局部行为精炼结合，解决了多智能体系统中的信用分配（Credit Assignment）难题。
*   **Token 效率：** 相比于 CORAG 等方法，HERA 在获得更高分数的同时，消耗的 Token 显著更少，证明了其“经验驱动”的精准性。
*   **可解释性：** 通过自然语言见解（Insights）而非数值梯度进行优化，使得系统的进化过程对人类开发者是可读、可调试的。

#### 8. 不足与局限
*   **推理延迟：** 虽然 Token 效率高，但在演化阶段（学习阶段）需要进行多组采样（Group Rollouts），这在初始阶段会带来较高的推理延迟和成本。
*   **特定能力缺失：** 案例分析显示，系统在处理需要复杂数值计算或集合运算（如时间区间交集）的任务时仍会失败，说明其缺乏专门的符号计算模块。
*   **对骨干模型的依赖：** 编排器的性能高度依赖于底层模型（如 Qwen-14B vs Llama-8B）的推理能力，弱模型可能无法生成高质量的语义见解。
