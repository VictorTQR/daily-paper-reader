## 摘要
智能体技能是程序性知识和可执行资源的结构化包，智能体在推理时动态加载这些技能，已成为增强大语言模型（LLM）智能体的一种可靠机制。然而，推理时的技能增强存在根本性的局限：检索噪声会引入无关的指导，注入的技能内容会带来巨大的 Token 开销，且模型从未真正习得它仅仅是在遵循的知识。我们探究技能是否可以内化到模型参数中，从而在无需任何运行时技能检索的情况下实现零样本自主行为。我们提出了 SKILL0，这是一个专为技能内化设计的上下文内强化学习框架。SKILL0 引入了一种训练时课程，从完整的技能上下文开始并逐步将其撤出。技能在离线状态下按类别分组，并与交互历史一起渲染成紧凑的视觉上下文，教导模型进行工具调用和多轮任务完成。随后，动态课程会评估每个技能文件的策略内（on-policy）帮助程度，在策略性衰减的预算内仅保留当前策略仍能从中获益的技能，直到智能体在完全零样本的设置下运行。广泛的智能体实验表明，SKILL0 相比标准强化学习基线取得了显著提升（ALFWorld 提升 9.7%，Search-QA 提升 6.6%），同时保持了每步少于 0.5k Token 的高效上下文。我们的代码可在 https://github.com/ZJU-REAL/SkillZero 获取。

## Abstract
Agent skills, structured packages of procedural knowledge and executable resources that agents dynamically load at inference time, have become a reliable mechanism for augmenting LLM agents. Yet inference-time skill augmentation is fundamentally limited: retrieval noise introduces irrelevant guidance, injected skill content imposes substantial token overhead, and the model never truly acquires the knowledge it merely follows. We ask whether skills can instead be internalized into model parameters, enabling zero-shot autonomous behavior without any runtime skill retrieval. We introduce SKILL0, an in-context reinforcement learning framework designed for skill internalization. SKILL0 introduces a training-time curriculum that begins with full skill context and progressively withdraws it. Skills are grouped offline by category and rendered with interaction history into a compact visual context, teaching he model tool invocation and multi-turn task completion. A Dynamic Curriculum then evaluates each skill file's on-policy helpfulness, retaining only those from which the current policy still benefits within a linearly decaying budget, until the agent operates in a fully zero-shot setting. Extensive agentic experiments demonstrate that SKILL0 achieves substantial improvements over the standard RL baseline (+9.7\% for ALFWorld and +6.6\% for Search-QA), while maintaining a highly efficient context of fewer than 0.5k tokens per step. Our code is available at https://github.com/ZJU-REAL/SkillZero.

### 1. 核心问题与研究动机
*   **核心问题**：当前的 LLM 智能体主要依赖“推理时技能增强”（即从技能库检索相关说明并放入 Prompt）。这种方式存在三大缺陷：
    1.  **检索噪声**：引入无关或误导性的指导。
    2.  **Token 开销**：多轮对话中注入技能描述会导致上下文长度激增，增加推理成本。
    3.  **缺乏真正习得**：模型只是在“遵循”指令，而非“掌握”知识，能力留在 Context 而非参数中。
*   **研究动机**：探索是否可以通过强化学习（RL）将外部技能“内化”到模型参数中，使智能体在推理阶段无需任何检索即可实现零样本（Zero-shot）的高效自主行为。

### 2. 方法论
SKILL0 提出了**上下文内强化学习（In-Context Reinforcement Learning, ICRL）**框架，核心思想是“训练时有技能，推理时零技能”。
*   **视觉上下文渲染**：为了降低 Token 成本，将交互历史和技能描述渲染为紧凑的图像，利用多模态编码器将其转化为视觉 Token。模型可以自适应生成压缩率 $c_t$。
*   **动态课程学习（Dynamic Curriculum）**：
    1.  **技能分组**：离线将技能按类别分组，并关联特定的验证子任务。
    2.  **帮助度评估**：定期计算技能的“策略内帮助度” $\Delta_k = Acc_{w/skill} - Acc_{w/o skill}$。
    3.  **线性预算衰减**：设置一个随训练阶段线性减少的技能数量预算 $M(s)$。
    4.  **过滤与排序**：在预算范围内，仅保留 $\Delta_k > 0$ 且排名靠前的技能。随着训练进行，预算最终降为 0，迫使模型将知识转移至参数。
*   **复合奖励函数**：奖励函数 $r_t$ 同时考虑任务成功率和 Token 压缩效率，通过 GRPO 等算法进行优化。

### 3. 实验设计
*   **数据集/场景**：
    1.  **ALFWorld**：具身智能体任务，包含 6 类家庭活动（如清洁、加热、放置物体等）。
    2.  **Search-based QA**：搜索增强问答，涵盖单步（NQ, TriviaQA, PopQA）和多步（HotpotQA, 2Wiki, MuSiQue, Bamboogle）推理任务。
*   **Benchmark 与对比方法**：
    *   **基线**：Zero-Shot、Few-Shot（带技能提示）。
    *   **RL 方法**：GRPO、AgentOCR（视觉压缩基线）、EvolveR。
    *   **技能增强方法**：SkillRL（推理时保留技能检索）。
    *   **模型规模**：基于 Qwen2.5-VL-3B 和 7B 展开实验。

*   **硬件**：使用了 **4 张 NVIDIA H800 GPU**。
*   **训练细节**：
    *   训练步数：最多 **180 步**。

    *   Prompt 长度限制：3072 到 4096 Token。

### 5. 实验数量与充分性
*   **实验规模**：论文在两个截然不同的领域（具身导航与知识问答）进行了大规模测试，验证了方法的通用性。
*   **消融实验**：非常充分。作者对比了不同的技能预算方案（如固定全量、固定低量、无技能训练）、不同的动态过滤策略（有无排序、有无过滤）以及验证间隔 $d$ 的影响。
*   **客观性**：通过在推理阶段完全移除技能提示，与带技能提示的方法进行公平对比，客观证明了“内化”的有效性。

### 6. 主要结论与发现
*   **性能卓越**：SKILL0 在零样本推理下，ALFWorld 成功率比 AgentOCR 提升 **9.7%**，Search-QA 提升 **6.6%**。
*   **极高效率**：推理时每步 Token 消耗**低于 0.5k**，相比 SkillRL 等依赖检索的方法，Token 成本降低了 5 倍以上。
*   **内化规律**：实验观察到技能的“帮助度”随训练呈现“先升后降”的趋势——初期模型学习利用技能，后期模型因知识内化不再依赖外部技能。

### 7. 优点
*   **范式突破**：成功实现了从“检索增强”到“参数内化”的转变，解决了智能体长期运行的 Token 瓶颈。
*   **动态自适应**：引入的帮助度评估机制比固定的课程表更科学，能根据模型实际掌握情况撤除“脚手架”。
*   **多模态融合**：巧妙利用视觉渲染解决长上下文问题，同时保持了结构化信息的完整性。

### 8. 不足与局限
*   **初始依赖**：内化效果高度依赖于初始技能库（SkillBank）的质量，若初始技能有误，模型可能内化错误的策略。
*   **冷启动成本**：在扩展到全新领域时，需要预先进行技能分组和验证集划分，存在一定的离线准备工作量。
*   **视觉损失风险**：虽然视觉渲染节省了 Token，但在极高压缩率下，细微的文本观察信息可能丢失，影响极高精度要求的任务。
