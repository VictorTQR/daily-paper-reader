## 摘要
随着基于大语言模型（LLM）的多智能体系统（MAS）从实验性试点演变为复杂且持久的生态系统，智能体之间直接通信的局限性日益凸显。当前的架构面临上下文碎片化、随机幻觉、僵化的安全边界以及低效的拓扑管理等问题。本文介绍了认知织网节点（Cognitive Fabric Nodes, CFN），这是一种新型中间件层，在智能体之间创建了一个无处不在的“认知织网”。与传统的消息队列或服务网格不同，CFN 不仅仅是透传机制，而是主动的智能中介。该架构的核心是将记忆（Memory）从简单的存储提升为主动的功能基质，从而支撑其他四项关键能力：拓扑选择、语义对齐（Semantic Grounding）、安全策略执行和提示词转换。我们建议这些功能中的每一项都由利用强化学习（RL）和优化算法的学习模块管理，以动态提高系统性能。通过拦截、分析和重写智能体间的通信，认知织网确保了单个智能体保持轻量化，同时使生态系统实现连贯性、安全性和语义一致性。我们在多智能体环境下的 HotPotQA 和 MuSiQue 数据集上评估了 CFN 的有效性，并证明与智能体间的直接通信相比，CFN 在这两个数据集上的性能提升均超过 10%。

## Abstract
As Large Language Model (LLM) based Multi-Agent Systems (MAS) evolve from experimental pilots to complex, persistent ecosystems, the limitations of direct agent-to-agent communication have become increasingly apparent. Current architectures suffer from fragmented context, stochastic hallucinations, rigid security boundaries, and inefficient topology management. This paper introduces Cognitive Fabric Nodes (CFN), a novel middleware layer that creates an omnipresent "Cognitive Fabric" between agents. Unlike traditional message queues or service meshes, CFNs are not merely pass-through mechanisms; they are active, intelligent intermediaries.   Central to this architecture is the elevation of Memory from simple storage to an active functional substrate that informs four other critical capabilities: Topology Selection, Semantic Grounding, Security Policy Enforcement, and Prompt Transformation. We propose that each of these functions be governed by learning modules utilizing Reinforcement Learning (RL) and optimization algorithms to improve system performance dynamically. By intercepting, analyzing, and rewriting inter-agent communication, the Cognitive Fabric ensures that individual agents remain lightweight while the ecosystem achieves coherence, safety, and semantic alignment.   We evaluate the effectiveness of the CFN on the HotPotQA and MuSiQue datasets in a multi-agent environment and demonstrate that the CFN improves performance by more than 10\% on both datasets over direct agent to agent communication.

### 2. 论文提出的方法论
CFN 的核心思想是将**记忆（Memory）**从被动存储提升为主动的“功能基质”，并以此支撑五大功能支柱：

*   **核心技术细节**：
    1.  **主动记忆（Active Memory）**：作为系统的“世界模型”，为所有操作提供上下文。
    2.  **拓扑选择（Topology Selection）**：将路由问题建模为**上下文多臂老虎机（Contextual Bandit）**问题。智能体广播“意图”而非指定接收者，CFN 根据历史表现、成本和负载动态选择最优执行路径。
    3.  **语义对齐（Semantic Grounding）**：通过计算消息与系统本体（Ontology）的余弦相似度，拦截或修正“鬼影实体”（不存在的对象）和语义偏差。
    4.  **安全策略执行（混合守护者）**：结合确定性规则（如正则过滤 PII）和基于强化学习的概率防御（检测对抗性提示词和级联信任攻击）。
    5.  **转换与重写（执行层）**：使用函数 $y_j = \Phi(x_{i \to j}, M, G, S | \theta)$ 对原始消息进行重构，注入上下文、脱敏并进行本体翻译。
*   **算法流程**：利用**强化学习（RL）**和 **TextGrad**（文本梯度下降）技术。系统通过评估任务结果生成自然语言反馈，并将其转化为“文本梯度”来更新 CFN 的重写策略 $\pi_{text}$，从而最小化“认知摩擦”。

### 6. 论文的主要结论与发现
*   **性能提升**：CFN 在 HotPotQA 和 MuSiQue 数据集上比直接通信模式性能提升了 **10% 以上**。
*   **接近上限**：CFN 优化后的 MAS 表现（91.5% / 86.1%）非常接近单模型在拥有完整信息时的表现（92% / 87.5%），有效解决了分布式环境下的信息衰减问题。
*   **认知税（Cognitive Tax）**：虽然中间件引入了额外的推理延迟，但通过减少智能体间的反复澄清和错误尝试，总体的“任务完成时间”反而可能下降。

### 7. 优点
*   **架构创新**：借鉴了微服务中的“服务网格（Service Mesh）”概念并将其升华为“认知织网”，实现了通信与逻辑的解耦。
*   **主动干预**：不同于传统中间件的透传，CFN 能够主动重写提示词，这种“语义网关”的设计对企业级安全和一致性至关重要。
*   **动态演进**：引入 RL 和 TextGrad 使得中间件具备自我进化能力，能根据智能体表现自动调整路由和对齐策略。

### 8. 不足与局限
*   **资源消耗**：在每个智能体旁部署 CFN（Sidecar 模式）会带来显著的计算资源开销（CPU/RAM）。
*   **同步延迟**：在跨地域的大规模分布式部署中，维持“逻辑中心化”的内存一致性面临 CAP 定理的挑战。
*   **任务单一**：目前的评估主要基于文本问答，对于多模态交互或需要实时物理反馈的场景（如机器人协作）尚未验证。
*   **依赖底层模型**：CFN 本身的重写质量高度依赖于其内置的小型语言模型（SLM）或 LLM 的推理能力。
